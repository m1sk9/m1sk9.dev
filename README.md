Personal portfolio and blog by [m1sk9](https://github.com/m1sk9), built with [Zola](https://www.getzola.org/) and the [Serene](https://github.com/isunjn/serene) theme (Serene has been partially customized).

## Prerequisites

- [Zola](https://www.getzola.org/documentation/getting-started/installation/) v0.21.0+
- [mise](https://mise.jdx.dev/) (optional, for version management)

## Development

```sh
# Install Zola via mise
mise install

# Start the dev server
zola serve
```

## Build

```sh
zola build
```

Output is written to the `build/` directory.

## security.txt

[`security.txt.in`](./security.txt.in) is the unsigned source of [RFC 9116](https://www.rfc-editor.org/rfc/rfc9116) `security.txt`. The signed file served at `https://m1sk9.dev/.well-known/security.txt` is generated by clearsigning it (requires the maintainer's YubiKey):

```sh
gpg -u 9700B737B52570D7B78394E52DF1A76B8DC2D6E3 \
    --clearsign --output static/.well-known/security.txt security.txt.in
```

Verify:

```sh
# Local
gpg --verify static/.well-known/security.txt

# Live
curl -s https://m1sk9.dev/.well-known/security.txt | gpg --verify
```

When the `Expires` date approaches, edit `security.txt.in` and re-sign.
